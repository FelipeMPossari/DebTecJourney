# MVP e Roadmap

## MVP

O primeiro MVP deve provar que a experiência principal funciona: aprender um conteúdo, responder perguntas, receber feedback e acompanhar progresso.

### Funcionalidades do aplicativo mobile

- Cadastro e login simples
- Tela inicial com trilhas de aprendizagem
- Lista de módulos por trilha
- Tela de lição com conteúdo curto
- Quiz de múltipla escolha
- Feedback imediato para acertos e erros
- Pontuação de XP
- Progresso por módulo
- Perfil básico do usuário

### Funcionalidades do backend

- Autenticação de usuário
- Consulta de trilhas, módulos, lições e perguntas
- Registro de respostas do usuário
- Cálculo de XP
- Registro de progresso
- Consulta de perfil e progresso

### Conteúdo mínimo

Para a primeira versão, recomenda-se criar uma trilha com 3 módulos:

1. Fundamentos da dívida técnica
2. Identificação e sintomas
3. Estratégias para lidar com dívida técnica

Cada módulo pode ter:

- 3 a 5 lições
- 3 a 5 perguntas por lição
- 1 cenário prático no final

## Roadmap sugerido

### Sprint 1 - Base do produto

- Definir identidade visual inicial
- Criar estrutura do app mobile
- Criar estrutura da API
- Configurar banco de dados
- Implementar entidades principais

### Sprint 2 - Conteúdo e progresso

- Criar endpoints de trilhas, módulos e lições
- Criar telas de listagem e leitura de lição
- Implementar progresso do usuário
- Popular banco com conteúdo inicial

### Sprint 3 - Quiz e gamificação

- Implementar perguntas e alternativas
- Implementar feedback imediato
- Registrar respostas
- Calcular XP
- Exibir progresso e pontuação

### Sprint 4 - Experiência e avaliação

- Melhorar interface
- Adicionar conquistas simples
- Criar fluxo de pré-teste e pós-teste
- Preparar coleta de dados para o TCC
- Ajustar usabilidade

## Critério de sucesso do MVP

Um usuário deve conseguir:

1. Criar conta ou entrar.
2. Escolher uma trilha.
3. Fazer uma lição.
4. Responder perguntas.
5. Receber feedback.
6. Ver seu XP e progresso salvos.
